# 三视角审查报告与修改计划

> 日期:2026-07-17 · 审查对象:`knowledgebase.md` v1.0、`related_solutions.md` v1.0
> 审查方式:三个独立视角并行——①事实与引用抽查(联网核对 30+ 项)、②一致性与需求覆盖(对照任务书逐条)、③可实施性与工程(架构师视角)。本文件为三份审查的合并与裁决。

---

## 总体结论

三个审查员的独立评价一致:**文档调研纪律与来源标注质量高于同类交付的常见水平**——抽查的高风险论文数字(Si et al. 全套、GPT-4o 78–90%、Undermind τ=80、Retraction Watch 计数、API 配额定价、10 个 GitHub star 数)全部逐字或量级命中;五套评分权重验算无误;14 条质量纪律 12 条确认遵守。

但存在三类实质问题:
1. **事实层**:3 处确定错误 + 3 处口径矛盾 + 9 处"该标未核实而没标";
2. **一致性层**:3 处节奏/时间线矛盾、机会框架字段分裂、related_solutions 修正未回写、1 个完全遗漏的需求点(用户上传论文);
3. **工程层**:文档是研究报告而非工程规格——"最后一公里"系统性缺失(学科体系未裁决、维度内公式空白、个性化排序双轨并存、OpenAlex 额度未做算术),规模叙事在千用户下崩塌,且缺"共享池 + snapshot + 历史回填"这组能一次性消解大部分矛盾的架构手段。

## P2:工程阻塞项(开工前必须裁决)

- **A1 学科体系裁决**:全文档"学科百分位"未指定分类体系 → 统一用 OpenAlex `primary_topic` 的 field/subfield 两级;一刊多类取最高百分位;分布从季度 snapshot 离线算成静态查表。
- **A2 评分维度内公式**:每维给显式聚合式 + 缺失值策略(缺失不记 0,降置信度并标注);扣分制量化档位(撤稿=一票否决,预警 venue 封顶 B 级等)。
- **A3 架构三件套(消解大部分规模矛盾)**:①**共享摄入湖**——按学科拉一次增量,所有用户档案本地匹配(放弃 per-user 查询,否则 1000 用户 $10/天击穿 OpenAlex 额度 10 倍);②**季度 snapshot 离线计算**引用切片与百分位分布;③**历史回填**——上线前用 snapshot 回算 36 个月主题指标,第一天即有趋势标签基线。
- **A4 个性化路线二选一**:裁决采用 Scholar Inbox 配方(80 万评分实证),"多质心向量层"废弃,由逻辑回归 + easy negatives 替代;同步改写 KB 11.2/11.3/12.2/17.1。
- **A5 人审模型重定义**:人审对象 = 学科共享卡片池(算一次审一次全员复用);个性化层只做自动门禁 + 抽检;引入用户 flag 机制。
- **A6 单点降级**:SCImago 条款待核实 + OpenAlex 指标自算作备用主路径;OpenAlex 失效的字段级降级矩阵;GS Metrics 佐证改为年度人工录入或删除。
