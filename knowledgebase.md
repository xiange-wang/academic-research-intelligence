## 1. 项目定义与目标

### 1.1 一句话定义

一个**面向多学科的学术研究情报系统**:用户声明一个研究方向(学科 / 方向 / 具体主题),系统持续采集该方向的新论文,评估其重要性与可信度,识别趋势与研究空白,并以**周报 / 月报 / 重点推荐**的形式交付。

### 1.2 核心能力(与需求逐条对应)

| # | 能力 | 本知识库对应章节 |
|---|------|----------------|
| 1 | 用户选择学科 / 方向 / 主题 | 第 11 章 |
| 2 | 持续检索最新论文 | 第 3、12 章 |
| 3 | 优先识别顶刊、顶会、高影响力机构成果 | 第 4、5 章 |
| 4 | 判断论文重要性、创新性、可信度、影响 | 第 6 章 |
| 5 | 总结主要方向、进展与争议 | 第 7、8 章 |
| 6 | 生成周报与月报 | 第 10 章 |
| 7 | 识别未解决问题与研究空白 | 第 9 章 |
| 8 | 推荐值得深读的论文与可研究的方向 | 第 6、9、10 章 |
| 9 | 跨学科适用 | 贯穿各章的"学科差异"小节 |

### 1.3 系统的角色定位(方法论层的三个基本立场)

1. **情报系统,不是评审系统。** 系统输出的是"值得关注的信号 + 证据",不是对论文的终审判断。所有评分都必须可解释、可追溯到具体特征。
2. **提示器,不是承诺器。** 研究机会输出的是"有证据支持的候选方向 + 不确定性说明",绝不将推测包装为确定可行的结论(详见 9.5、15 章)。
3. **多信号融合,不迷信任何单一指标。** 影响因子、引用数、venue 等级都只是信号之一;系统必须显式对抗"单指标霸权"(DORA / Leiden Manifesto 的核心要求,详见第 5 章)。

## 2. 系统适用范围与边界

### 2.1 适用范围

- **学科**:目标覆盖 STEM + 医学 + 社会科学;人文学科(依赖专著、非英语文献)列为受限支持,在报告中显式声明覆盖不足。
- **文献类型**:期刊论文、会议论文、预印本、综述 / 系统综述、指南与共识声明、数据集 / 基准论文。专著、学位论文、专利列为二期扩展。
- **语言**:一期以英语文献为主,元数据层面兼容多语言;中文文献源(CNKI 等)列为二期评估项。

### 2.2 明确不做(边界)

1. **不提供论文全文的分发**——只存储与展示元数据、合规获取的摘要、开放获取全文的链接与解析结果(版权边界详见第 16 章)。
2. **不替代同行评审**——不输出"这篇论文是错的"这类终审结论,只输出可核查的信号(撤稿记录、争议引用、方法学红旗)。
3. **不做全网爬虫**——数据获取以官方 API 与开放数据集为主,不绕过出版商访问控制。
4. **不承诺研究机会的可行性**——机会条目一律附带证据强度与风险标注。

### 2.3 关键设计约束(对后续技术选型的约束,非选型本身)

- 所有采集必须支持**增量更新**(按日期游标拉取),这是数据源取舍的硬条件。
- 每条结论(评分、趋势判断、空白识别)必须保留**证据链**:来自哪些论文、哪些字段、哪个规则或模型。
- 学科参数(评分权重、venue 分级映射、趋势阈值)必须是**配置而非硬编码**,因为学科差异贯穿所有模块。
- **共享摄入湖架构**(v1.2 裁决,v1.3 补例外白名单):按学科拉取一次增量,所有用户档案在本地匹配;**禁止 per-user 持续性对外查询**——成本算术:1000 用户 × 10 个持久化子查询/天 × $0.001(OpenAlex search)= $10/天,击穿免费额度 10 倍;而共享增量(日均新增约 2.5–3 万条,~150 次 list 调用)仅 ~$0.015/天。**例外白名单(一次性/低频操作,各设预算上限)**:①冷启动种子论文导入;②空白识别的检索验证(9.6 Owl 范式);③评测用独立检索采样(17.2);④档案超出已建学科湖范围时的一次性补召回(并触发新学科建湖评估)。
- **多租户演进路径**:单用户形态(个人部署、单档案)→ 千用户形态(共享摄入湖 + 共享卡片池 + 档案隔离);两形态共用同一数据模型,演进只改部署拓扑不改 schema——这是选型阶段的硬约束。
- **重计算走季度 snapshot 离线做**:引用切片、学科百分位分布、趋势指标回算一律基于本地 snapshot(OpenAlex 免费 snapshot 为季度更新),API 只承担当日新增。
- **数据规模量级**(供选型):全学科日增 ~2.5–3 万条元数据;单学科 36 个月全量约百万级摘要嵌入;向量检索与图查询(论文/作者/机构/venue/主题五类节点)是硬需求(源自 12.2)。
- **成本包络**:参考实证——LLM 筛选 $0.07/天(全 cs.CL)、DeepSeek 级日摘要 ~0.2 元/天;端到端成本表(单方向/周与 1000 用户两档)在选型前按 review_report 实验 3 补齐。
- **用户数据归属**:反馈与主题档案数据归用户所有、可导出迁移(订阅与评分数据是可迁移资产);档案间与用户间隔离;兴趣数据不得用于跨用户推断(除非显式同意)。
- **中国部署合规**:无 TDM 法定例外(见 16.2),境内部署依托 API 授权与 CC0/OA 许可,不依赖法定例外;重建摘要仅限内部处理。

## 3. 学术数据源地图

> 调研时间:2026 年 7 月,以官方文档为准。所有未能从官方页面复核的数字均标注"未核实"。3.1–3.3 为开放/免费数据源;3.4 为商业数据库与出版商 API;3.5 为竞品产品全景。

### 3.1 开放数据源逐项档案

#### 3.1.1 OpenAlex —— 推荐的多学科主源

- **地位**:Microsoft Academic Graph(2021 年底停运)的公认继任者,由 OurResearch 于 2022 年 1 月发布,现为开放学术元数据的核心基础设施。
- **覆盖**:约 **3.2 亿 works**(API 实测 2026-07-17:320,560,176)、90M+ 作者、100K+ 来源(作者/来源数为官方文档口径,快照时点未注明),全学科。
- **API**:`api.openalex.org`。**2025–2026 重大变化**:文档迁移至 developers.openalex.org,引入免费 API Key + 基于成本(credit)的限流模型——持 key 每天 $1 免费额度(list 查询 $0.0001/次、search $0.001/次、内容下载 $0.01/次、单条查询免费);无 key 每天仅 $0.01。
- **付费**:Premium 提供更高限额、月度快照、**每日变更文件**(daily change files——注意:每日增量文件现属付费功能),价格未公开(未核实)。
- **数据质量**:引文关系、倒排索引摘要、作者/机构消歧(ORCID、ROR)。许可 **CC0,可商用**。
- **增量**:支持按主题/关键词/日期过滤(`from_created_date`/`from_updated_date` 参数名以最新官方文档为准)。
- **对本系统的含义**:免费额度下做全学科每日增量需要精细预算;或接受月度快照 + 免费 API 补拉的混合模式。

#### 3.1.2 Crossref —— DOI 权威源

- **覆盖**:184,579,686 works(2026-07 实时计数),所有注册 DOI 的出版物。
- **API**:`api.crossref.org`,免注册;`mailto=` 参数进入礼貌池;速率经响应头动态告知(固定数字未核实)。付费层 Metadata Plus(快照 + SLA)。
- **数据质量**:funder、ORCID、ROR、参考文献(取决于会员开放);摘要覆盖不全且部分受出版商版权;作者消歧弱。
- **许可**:元数据基本不受版权限制;**部分摘要例外**。
- **增量**:`from-index-date` + cursor 深分页,非常适合增量抓取。

#### 3.1.3 PubMed / Europe PMC —— 生物医学双主源

- **PubMed(NCBI E-utilities)**:40M+ 条记录;免费;无 key 3 req/s、有 key 10 req/s;**每日**更新文件;有 MeSH 主题词;本体无引文数据。增量检索是标杆级的:`datetype=edat&reldate=1` 即"昨日新收录"。注意:NLM 不主张摘要版权,但出版商/作者可能持有。
- **Europe PMC**:48.4M 条,整合 PubMed、PMC、Agricola 及 34 个生命科学预印本服务器;免费 REST API,无需认证;**每日更新**;自带引文网络(smart citations)、ORCID 集成、OA 全文子集;`FIRST_PDATE` 日期过滤。速率上限官方未明示(未核实)。
- **对本系统的含义**:医学/生物方向的监控以这两者为主,PubMed 管"新收录信号",Europe PMC 补引文与全文。

#### 3.1.4 预印本服务器

- **arXiv**:2026 年 4 月突破 300 万篇(7 月约 3,099,360);物理/数学/CS 为主。API 免费无认证,但**限速 1 请求/3 秒**、单次 ≤2000 条;OAI-PMH(`oaipmh.arxiv.org`)与分类 RSS(`rss.arxiv.org/rss/<cat>`)适合每日监控;公告节奏为周日至周四(美东约 20:00)。**元数据 CC0,全文不得转储再分发**。无引文数据,机构字段质量一般。已于 **2026-07-01 正式完成独立非营利转制**(arXiv 官方博客 2026-06-30/07-09)。
- **bioRxiv / medRxiv**:合计 38.9 万+ 预印本;共用 `api.biorxiv.org`,免费无认证;`details` 端点按日期区间 + 游标分页(每次 30 条);**API 无关键词过滤**,需拉回后客户端过滤;许可逐篇(CC BY 至保留所有权利不等)。2025 年 3 月起由独立非营利 **openRxiv** 运营。
- **SSRN**:社科工作论文重镇,但 **无官方公开 API**(Elsevier 所有),仅能靠 eJournal RSS 近似监控——社科覆盖的一个结构性缺口,需在报告覆盖声明中显式披露。

#### 3.1.5 引文与书目专用源

- **OpenCitations**:开放引文基础设施,数据 CC0;REST API 限速 180 req/min;按 DOI/OMID 查引文关系,非发现型检索。用途:为主源交叉补引文。
- **Semantic Scholar**:Graph/Recommendations/Datasets 三套 API;提供 TLDR 自动摘要、SPECTER2 论文嵌入、influential citation 标注。**政策收紧(2025–2026)**:持 key 入门仅 1 RPS(人工审核后可提);无 key 共享约 1000 RPS 全局池,高峰降级。数据集许可多为 ODC-BY/带条款,商用需逐条核对(未核实)。定位:小批量富化(嵌入/TLDR/引文),不适合作高频主抓取源。
- **DBLP**:CS 书目权威,CC0;搜索 API 免认证 + RSS + 全量 XML 转储;以作者消歧著称;**无摘要、无引文**。用途:CS 方向的 venue/作者规范化与去重基准。

#### 3.1.6 聚合与目录

- **CORE**:约 4.31 亿元数据记录(未核实)、数千万自托管全文;API v3 需 key,免费层速率低(约 5 次/10 秒,未核实);数据转储 ODC-BY。harvest 为事件驱动、非每日——**适合建全文语料,不适合当日新刊监控**。
- **DOAJ**:OA 期刊目录(约 2.1 万刊,未核实);读 API 免费;无引文。用途:期刊白名单信号之一(注意 DOAJ 除名名单也是预警信号,见第 4 章)。
- **The Lens**:200M+ 学术记录 + 大规模专利库;API 需 token,免费限非商业,**商用需付费订阅**(价格未公开);特色是学术↔专利关联,可服务于"产业影响"信号。

#### 3.1.7 已停运/需替代

- **Papers with Code:已停止独立运营**(2026-07 实测 302 重定向到 huggingface.co/papers/trending;由 Meta 于 2025 年前后归档)。"论文↔代码/模型"关联信号改用 **Hugging Face Papers**;历史数据(CC BY-SA)仍可作训练/回填。

### 3.3 持续监控的推荐源组合(开放部分)

- **多学科主干**:OpenAlex(元数据 + 引文 + 消歧)⊕ Crossref(DOI 权威 + 增量索引)——以 DOI + OpenAlex ID 为主键跨源去重。
- **生物医学**:PubMed(每日新收录)⊕ Europe PMC(引文/全文/预印本聚合)⊕ bioRxiv/medRxiv(预印本)。
- **CS / 数学 / 物理**:arXiv(每日 RSS + OAI-PMH)⊕ DBLP(venue/作者规范化)⊕ Hugging Face Papers(代码模型关联)。
- **富化层**:Semantic Scholar(TLDR/嵌入/influential citations,低速批量)⊕ OpenCitations(引文交叉验证)。
- **已知缺口**:SSRN 无 API(社科工作论文监控受限);OpenAlex 每日变更文件付费;人文学科覆盖普遍薄弱。这些缺口写入报告"覆盖声明"。
- **单点降级预案(v1.2)**:OpenAlex 是全系统最大单点,其失效/涨价时——DOI 与出版元数据由 Crossref 承接、引用关系由 OpenCitations/S2 部分承接,但 **topic 挂载与 `counts_by_year` 无等价替代**(相关能力优雅退化为"仅当期计数");SCImago CSV 失效时以 OpenAlex 指标自算分位替代;Hugging Face Papers 仅"锦上添花",代码可用性主路径为自建 arXiv 正文链接抽取(见 6.2)。撤稿数据已有三源交叉(15.1),作为降级设计的模板。

### 3.4 商业数据库、出版商 API 与发现工具

### 4.4 学科差异的证据链

- **CS 重会议**:CSRankings 官方 FAQ 明确只按顶会发表计量、拒绝引用指标("repeatedly shown to be subject to manipulation")与声誉问卷;数据源 DBLP,全开源。

## 5. 跨学科学术质量评价框架

### 5.1 负责任科研评价的合规约束(框架的"宪法")

- **DORA(2012,27,259 个签署方)**:第 1 条——不得以期刊指标(如 JIF)作为单篇论文质量或个人贡献的替代度量。
- **Leiden Manifesto(Nature 2015, DOI:10.1038/520429a)**:10 条原则,对本系统最关键的是:#4 数据与分析过程公开透明简单、#5 允许被评者核对、#6 按学科规范化、#8 避免虚假精确、#10 定期重审指标。
- **CoARA(2022,800+ 组织)**:承诺 3——放弃期刊/出版物指标(点名 JIF、h-index)在科研评价中的不当使用。

**落地为六条系统规则**:① 分级仅用于"投稿目标筛选 / 情报聚合",禁止换算为个人绩效;② 多指标并列展示原始值,不造单一黑箱总分;③ 一切引用指标只在学科内取百分位;④ 判定规则与数据快照公开可复核、可申诉;⑤ 分级边界 ±2 百分点内标"边界案例"转人工;⑥ 年度重审阈值,公开 changelog(警惕 Goodhart 效应)。

### 5.2 五级分级框架(S / A / B / 普通 / 预警)判定规则

**学科体系裁决(v1.2)**:全框架(含第 6、7 章)的"学科百分位"统一以 **OpenAlex `primary_topic` 的 field/subfield 两级**为准;一刊多类时取最高百分位定级(与 5.3 交叉学科规则一致);百分位分布由**季度 snapshot 离线全量计算**产出静态查表(不做在线计算);JCR/SCImago 学科类别仅作映射对照,映射表随分级表年度维护。

规则**自上而下短路执行**;每个判定输出 `级别 + 触发规则 ID + 原始指标值 + 数据快照日期`。

**红旗语义(v1.3 统一)**:红旗信号以 **4.3 的唯一清单(R1–R7)** 为准,本节只定义触发语义——**预警触发**:命中 R1、R2、R7 任一,或 R3–R6 中 ≥2 项;**S/A 排除**:任何 1 项红旗即排除 S/A 级,并触发降级规则(见下)。**降级规则(修复"掉级黑洞")**:有红旗但未达预警触发的 venue,先按无红旗规则定级,再**降一级并标注红旗 ID**(如 Q1 顶刊带 1 项红旗 → A 级 + 红旗标注),不因短路顺序静默落入普通级。

**第 0 步 · 预警级(一票否决)**——按上述预警触发条件判定。→ 输出理由码,允许申诉。

**S 级(旗舰,学科前 ~2–5%)**——"分区/评级"与"独立佐证"各至少 1 项且无红旗:
- 期刊路径:JCR Q1 且 JIF 学科百分位 ≥95(无 JCR 时:SCImago Q1 且 SJR 百分位 ≥95),**且**独立佐证至少 1 项——OpenAlex `2yr_mean_citedness` 学科百分位 ≥90,或 `h_index` 学科前 5%,或 GS Metrics 领域榜前 20(GS 无 API,每年 Metrics 发布后人工录入一次);
- 会议路径(v1.3 澄清):**CCF-A 或 ICORE A\* 单条件即满足 S 级**(免独立佐证——现有佐证项均为期刊指标口径,对会议不适用;GS Metrics 领域榜可作可选加强信号),无红旗即定 S。

**A 级(优秀,前 ~5–20%)**:期刊 Q1 未达 S 佐证或百分位 80–95;会议 CCF-B 或 ICORE A;OpenAlex 百分位 ≥75 佐证;无红旗。

**B 级(良好,前 ~20–50%)**:期刊 Q2 或百分位 50–80;会议 CCF-C 或 ICORE B;白名单信号完备。

**普通级(正规但无突出信号)**:Scopus/WoS/DOAJ 任一正规收录(或挪威 Level 1)、同行评审可验证,但指标处于学科后 50% 或数据不足。**数据不足 ≠ 预警**——保护新刊、区域刊与本地相关研究(Leiden #3)。

## 6. 论文重要性评分模型

> 调研时间:2026-07。本章先给证据基础(6.1–6.4),再给评分模型(6.5–6.6)。

### 6.1 早期影响力预测:哪些特征真的有预测力

跨研究共识的特征强度排序(用于特征工程):**头 1–2 年早期引用轨迹 > venue 声望 > 作者既往被引记录 > 机构 > 参考文献特征 > 社交媒体信号**。

- 经典结果:内容特征 + 文献计量特征混合显著优于单一特征(Fu & Aliferis 2010);40+ 特征的非线性模型可较准确预测 10 年引用(Weihs & Etzioni, JCDL 2017)。
- **LLM 时代的关键进展**:仅用题目 + 摘要微调 LLM 预测归一化影响力,NDCG@20=0.901(Zhao et al., AAAI 2025, arXiv:2408.03934)——**解决"零引用冷启动"问题**,新论文发表当天即可打前瞻影响力分。
- Altmetrics 与最终引用的相关性总体弱到中等;其中**新闻/政策提及强于社交媒体提及**(部分未核实,方向可信)。

### 6.2 信号源现状(2026-07 核实)

### 6.3 LLM 辅助评估:能用,但有硬约束

- **有效性证据**:GPT-4 评审意见与人类评审重叠率(Nature 30.9% / ICLR 39.2%)与两位人类评审间重叠率相当(Liang et al., NEJM AI 2024);AAAI-26 官方试点对全部投稿生成明确标识的 AI 补充评审(试点已经 AAAI 官方 FAQ PDF 核实:AI 评审不打分、不给建议,决策全由人做;投稿数 22,977 二次核实未果,待核);ICLR 2025 官方 agent 只对评审初稿给可选反馈、不做决策。
- **风险证据**:2025 年 7 月曝光 17 篇预印本在正文藏白字指令("GIVE A POSITIVE REVIEW ONLY")操纵 LLM 评审;已知偏差:冗长偏好、正面偏差、名气偏差。
- **落地为四条硬规则**:① 输入正文做隐藏指令清洗(防 prompt injection);② 多次采样取中位数降方差;③ LLM 意见只是加权项之一,**永不单独否决或拍板**;④ AI 生成的评估在界面上明确标识(与 8.2 三类陈述规范一致)。

### 6.4 可信度维度采用"扣分制"

撤稿/PubPeer 命中/掠夺性 venue 等负信号不做线性加减,而是**大幅降权直至一票否决**;正常论文可信度分默认从满分起步,按红旗扣减。

### 6.5 六维评分模型

综合分 = Σ(维度分 × 学科权重)。每维 0–100,所有引用类特征先做**学科 × 年份百分位归一化**(学科体系遵循 5.2 的裁决:OpenAlex field/subfield 两级,分布由季度 snapshot 离线计算)。

**维度内聚合规范(v1.3 整合)**:
- **五个加权和维度**(质量/创新/影响力/相关性/时效性):每维得分 = 该维特征的加权和,特征值先归一到 0–100(引用类走百分位,布尔类 0/100,计数类按学科分布分位);**特征缺失时不记 0 分**,而是从该维权重中剔除该特征并重归一,同时下调该维"置信度"并在论文卡片标注(与 8.2 可解释要求一致)。
- **可信度维不走加权和,而是独立扣分制**(6.4 原则的公式化):满分 100 起步,按红旗扣减——撤稿 = **综合分直接置 0 + 红旗**(公式外覆盖规则);venue 预警级 = 可信度维 ≤20 且推荐档位封顶"收录可查";PubPeer 实质性质疑 = −30 并转人工;预印本(未同行评审)= −10 并标注。8.1C 的可信度信号清单与扣分项一一对应,**每个信号只在扣分制中计一次**(不再作为加权特征重复计入)。以上均为设计初值,按 12.3 回测校准。

**与排序层的分工(v1.3 裁决,消除双轨)**:周报内的**排序顺序**由 11.2 排序层(每档案逻辑回归)决定——其输出即本模型"相关性维"的实现,**不重复计算**;六维综合分负责**准入与呈现**(是否入选、进哪一档、卡片上如何解释)。即 `排序 = LR 分;入选与展示 = 六维分(其中相关性维直接引用 LR 分)`。

| 维度 | 可计算特征 | 数据源 | 基线权重 |
|---|---|---|---|
| **质量** | SciScore 类方法学检测、statcheck 统计一致性、venue 分级(第 5 章)、注册报告标志、样本量/预注册 | SciScore、statcheck、OpenAlex、OSF | 20% |
| **创新** | 摘要 embedding 与近期文献的距离(新颖度)、参考文献组合非常规性(Uzzi atypicality)、LLM 创新性初判(受 6.3 约束) | 自建 embedding、引文图 | 15% |
| **影响力** | 学科内引用百分位、引用增速(`counts_by_year` 斜率)、influential citations、**冷启动:题目+摘要 LLM 预测分** | OpenAlex、S2、自建模型 | 20% |
| **可信度** | 撤稿/关注表达、PubPeer 命中、预印本 vs 正式版、venue 预警状态、作者/机构既往撤稿(扣分制) | Retraction Watch、Crossref、OpenAlex | 20% |
| **相关性** | **= 11.2 排序层逻辑回归的输出**(其特征已含嵌入相似度、种子/收藏信号);另加符号层命中(关注作者/论文被引用、主题标签匹配)作加分项 | 排序层(11.2) | 15% |
| **时效性** | 发表时间衰减、所属主题热度斜率(第 7 章)、早期关注(Bluesky/新闻/政策) | 日期、自建 altmetrics | 10% |

**可选加分项**:代码/数据可用性("声明有"与"实际可访问"分开核验)→ 并入质量或影响力。

## 7. 最新研究趋势识别方法

> 调研时间:2026-07,关键论断均经原始文献核验(来源见 7.7)。

### 7.1 方法地基:四大文献计量范式的取舍

| 方法 | 奠基文献 | 可即时计算 | 时滞 | 在本系统中的角色 |
|---|---|---|---|---|
| 共引分析 | Small 1973 | 否(需被引积累) | 高(2–5 年) | 只用于领域知识基础回顾,不用于趋势 |
| **文献耦合** | Kessler 1963 | **是**(发表当天可算) | 低 | **研究前沿聚类的引用侧首选** |
| 关键词共现 | Callon et al. 1983/1991(战略坐标图:向心度×密度四象限) | 是 | 低 | 主题结构与演化;现代做法用 NLP 短语替代人工标引词 |
| 合作网络 | Newman 2001 | 是 | 低 | 社区形成、新作者流入信号 |

**关键实证**:Boyack & Klavans 2010(215 万篇生物医学论文):文献耦合略优于共引、直接引用最差、**引用+文本混合方法全面最优**——本系统采用混合路线(嵌入聚类为主、耦合为辅)。

### 7.2 突发检测(Kleinberg burst detection)

- **原理**(Kleinberg KDD 2002):事件流建模为两状态自动机(基线态/突发态),突发率 p₁ = s×p₀(s 控制门槛),状态跃迁付出与 γ 成正比的代价(抑制噪声跳变),Viterbi 式动态规划求最优状态序列;burst weight 度量强度。实现:R `bursts` 包、Python `burst_detection`。
- **CiteSpace**(Chen 2006)将其推广到多词术语与文献被引序列(keyword burst / citation burst);VOSviewer **不支持** burst。
- 组合用法:Kleinberg burst(检测突变)+ **Mann–Kendall 检验 + Sen's slope**(检测持续单调趋势)互补(Scientometrics 2024 有范例)。

### 7.3 主题建模选型

- **实证对比**(Egger & Yu 2022, Frontiers in Sociology):BERTopic 最佳,NMF 次之,LDA 最弱(产出泛而无关的主题)。警示:嵌入类方法主题数多、HDBSCAN 产生大量 outlier、UMAP/HDBSCAN 有运行间随机性。
- **当代标准做法**:**SPECTER2 科学文献嵌入(引用图信号预训练,Cohan et al. 2020)+ UMAP + HDBSCAN + c-TF-IDF**(即定制化 BERTopic);LDA 仅作基线。评估用主题一致性 C_v/NPMI(Röder 2015)+ 人工抽检。
- **嵌入选型取舍(v1.2)**:个性化排序场景实测 **GTE-Large(降 256 维)> SPECTER2**(Scholar Inbox 消融,见 related_solutions 3.1),且 SPECTER2 至今无官方后继;聚类场景 SPECTER2 的引文信号仍有价值。两场景嵌入是否统一,以小基准实验裁决(review_report 实验 7);可商用候选:gte-large(MIT)、Qwen3-Embedding(Apache-2.0);**NV-Embed-v2 为 CC-BY-NC 不可商用**。
- **对本系统关键的 BERTopic 能力**(官方文档确认):`topics_over_time()`(动态主题)、`partial_fit()`(增量)、`hierarchical_topics()`、`merge_topics()`——天然支持"增量摄入 + 时序演化"。

### 7.4 新兴主题的操作性定义

领域公认框架收敛于四个可计算维度:**novelty(新颖)+ growth(增长)+ coherence/persistence(凝聚且持续)+ impact/community(影响与社区)**:

- Rotolo, Hicks & Martin 2015(Research Policy):新兴技术五属性——根本新颖、相对快速增长、连贯性、显著影响、不确定性(最后一项作解释性维度)。
- Small, Boyack & Klavans 2014:以"新出现的聚类"与"快速增长的聚类"操作化 emergence。
- Porter et al. 2019:可计算 emergence score;Wang 2018(JASIST):novelty + growth + coherence + 科学影响。

### 7.5 增长指标与先行信号

- **全科学基线增长率:4.10%/年,倍增周期 17.3 年**(Bornmann et al. 2021, 已核验)——判断"某主题增长是否跑赢大盘"的分母。
- **先行信号排序**(仪表盘按此分层,禁止用滞后信号否决先行信号):**基金(先行 2–3 年)→ 预印本(先行约 6 个月,bioRxiv 预印本→期刊发表中位间隔 166 天,Abdill & Blekhman 2019)→ 期刊论文 → 引用(滞后 1–3 年)**。
- 基金数据可编程获取:NIH RePORTER API v2(全文检索 + 金额,约 1 req/s)、NSF Awards API(每页 25 条、单查询上限 3000 条、2007 年起)。**覆盖声明:基金信号目前仅覆盖美国**,对非美主导领域会系统性失真;ERC/NSFC 数据可得性列为二期评估。
- 其他先行信号(二期扩展):产业实验室技术报告(企业研究院发布流)、专利—论文关联(The Lens,见 3.1.6);顶会 CFP/专题变化并入周报"领域动态"事件流。
- **反例警示**:睡美人现象并不罕见(Ke et al. 2015, PNAS, 2200 万篇论文)——引用类指标对 <18 个月的主题不作硬条件。

### 7.6 炒作识别(有实证支撑的信号)

- **石墨烯指纹**(Klincewicz 2016):论文量指数增长的同时,**篇均引用与高被引占比下降**(数量跑赢质量)——核心判别量 `cpp_trend`。
- **语言 hype**:PubMed 摘要积极词汇 40 年 +880%(Vinkers et al. 2015, BMJ);NIH 申请书 hype 形容词 +1378%(Millar 系列,139 词表);hype(促销性语言)自动检测已被形式化为分类任务,微调 BERT 达 **86.1%**,超人类基线 82.4%(arXiv:2509.24638,已核实:全文 Table 5)——词表与分类器思路可复用。
- **关注度—影响脱节**:Altmetric 与引用相关性弱;媒体声量远超引用增长是炒作信号。
- **Proteus 现象**(Ioannidis 2006):早期夸大主张与极端反驳快速交替——争议标签的理论依据。
- Gartner hype cycle 本身实证支持有限(Dedehayir & Steinert 2016),不直接套用曲线,只借用"期望峰—幻灭"动态。

### 7.7 趋势识别流程(可操作交付)

```
论文流(OpenAlex/S2/arXiv/PubMed)+ 基金流(NIH/NSF)
 → [1]摄入清洗 → [2]SPECTER2 嵌入 → [3]聚类(UMAP+HDBSCAN/BERTopic)
 → [4]主题级七组指标 → [5]规则引擎打六类标签(可复合) → [6]人工核查视图
```

**最小数据字段**:paper_id、title+abstract、pub_date(到月)、references、authors+affiliations 为必需;citation 时间切片(OpenAlex `counts_by_year`)、venue+doc_type 强烈建议;撤稿状态、基金号、Altmetric 可选。

**七组主题级指标**(每主题 × 每时间窗):

| 组 | 核心指标 |
|---|---|
| A 增长 | 相对增长(CAGR÷领域基线)、Kleinberg burst(s=2, γ=1 起步)、Mann–Kendall+Sen's slope、基金增长、预印本增速 |
| B 新颖 | 主题年龄、质心新颖度(1−max 历史相似度)、新术语比例、参考文献 Price 指数 |
| C 凝聚 | 聚类内平均余弦 + 主题词 NPMI、跨窗口存活数(persistence) |
| D 影响 | 引用速度/加速度、主题外引用占比、**篇均引用趋势 cpp_trend** |
| E 社区 | 新作者占比、机构多样性、活跃作者增速 |
| F 炒作 | hype 词表 z 分数、Altmetric/引用比、综述占比增速、低质 venue 混合度 |
| G 争议 | comment/letter/erratum 占比、撤稿率、结论方向震荡度 |

**六类标签判定**(v1.3 改为两段式,消除漏判与顺序歧义):**第一段并行评估**全部六类标签的条件;**第二段按优先级消解冲突**——主标签优先级:衰退 > 新兴 > 快速增长 > 成熟,争议与疑似炒作为**修饰标签**,可叠加在任何主标签上(含衰退);**任何主标签都不满足时输出"观察中"缺省态**(不强行归类,等待数据)。"已过冷启动"定义为:主题年龄 >3 年或体量 ≥P25。阈值说明:增长/引用/指标类阈值用领域内分位数(P75/P90 等,参照总体见"冷启动与历史回填"段);年龄、占比类(≤3 年、≥0.5、<0.25、CAGR≥20%)为**有意的绝对阈值**(跨领域可直读);均每半年用已知案例回测重校:

| 标签 | 判定要点 |
|---|---|
| **新兴** | 年龄 ≤3 年 且 novelty≥P75 且 相对增长 ≥2× 基线 且 coherence≥P50 且存活 ≥3 个月度窗口 且新作者占比 ≥0.5;引用不作硬条件 |
| **快速增长** | 已过冷启动,相对增长 ≥2× 基线(或 CAGR≥20%),burst 活跃或 MK 显著上升,且 **cpp_trend≥0**(排除纯数量膨胀) |
| **成熟** | 体量 ≥P50 且增速与基线差 <5pp 持续 ≥2 年,参考文献变旧,新作者占比低位稳定 |
| **衰退** | 论文量与引用速度**双降**且 MK 显著下降 ≥2 年、新作者 <0.25 且降、基金负增长;需排除"主题改名漂移" |
| **争议** | 活跃且满足 ≥2 项:dispute_ratio≥2× 中位、撤稿/重复失败事件、结论震荡、高被引论文遭显著负面引用;触发即转人工 |
| **疑似炒作** | 相对增长 ≥2× 基线 且满足 ≥3 项:cpp_trend<0、hype 词 ≥P90、Altmetric/引用比 ≥P90、综述增速 ≥2× 论文增速、venue 质量恶化、novelty 低;标签仅为"疑似",必须人工复核 |

**增量节奏**:每周——增量拉取、嵌入归入既有主题(不改标签,只刷新 burst 告警,12 周 EWMA 平滑出版批次噪声);每月——重算全部指标(36 个月滑窗)、更新标签并输出 diff、检查 outlier 池(占比 >15% 或 ≥20 篇自聚类达标 → 注册候选新主题);每季/半年——全量重聚类 + Hungarian 匹配对齐新旧主题、继承标签历史、重校阈值。

**冷启动与历史回填(v1.2,消解数据积累矛盾的关键手段)**:上线前用季度 snapshot **一次性回算目标学科近 36 个月的全部主题指标**(除用户反馈类外,7.7 的指标均可回填),使六类趋势标签在上线第一天即有基线;"成熟/衰退"等需 ≥2 年窗口的标签由回填数据支撑。回填结果同时用于:在已知案例(如 2012–2016 深度学习=新兴→快速增长、2010–2015 石墨烯=后期疑似炒作)上验证标签复现效果(review_report 实验 5)。阈值分位数的参照总体 = 学科级全量主题(OpenAlex topics 按 field 分组),同样从 snapshot 离线计算成静态查表。

**必须处理的系统性偏差**:① 索引回填滞后(近 2–3 个月覆盖不全,增长率需用同期完整度曲线校正,或只对 ≥3 个月前数据下结论);② 引用时滞与睡美人;③ 先行信号分层展示。

**工具定位**:可编程流水线用 Python(litstudy / BERTopic / burst_detection / pymannkendall,均开源)或 R(bibliometrix, GPL-3);CiteSpace / VOSviewer 只作为**人工核查与可视化终端**,不进流水线。

**核心来源**:Small 1973 (DOI:10.1002/asi.4630240406); Kessler 1963; Boyack & Klavans 2010 (DOI:10.1002/asi.21419); Kleinberg 2002/2003; Chen 2006 (CiteSpace II); Grootendorst 2022 (arXiv:2203.05794); Egger & Yu 2022; Cohan et al. 2020 SPECTER (arXiv:2004.07180); Rotolo et al. 2015 (DOI:10.1016/j.respol.2015.06.006); Bornmann et al. 2021; Ke et al. 2015 (PNAS); Abdill & Blekhman 2019 (eLife); Klincewicz 2016; Vinkers et al. 2015 (BMJ); Millar et al. 2022 (JAMA Netw Open); Ioannidis 2006 (Proteus)。

## 8. 重点论文总结标准

### 8.1 论文卡片(Paper Card)字段规范

每篇进入周报 / 月报 / 推荐列表的论文,生成一张结构化"论文卡片",分为四个区:

**A. 书目区(纯元数据,机器填充,不允许生成式改写)**

| 字段 | 来源 | 备注 |
|------|------|------|
| 标题 / 作者 / 机构 | 数据源元数据 | 机构需消歧(ROR ID) |
| 发表时间 | 元数据 | 区分:预印本首发时间 / 正式发表时间 |
| 期刊或会议 + 本系统分级 | 元数据 + 第 5 章分级框架 | 标注分级依据 |
| DOI / 官方链接 | 元数据 | 优先 DOI,预印本给 arXiv/bioRxiv ID |
| 论文类型 | 分类器 + 元数据 | 原始研究 / 综述 / 系统综述 / 指南共识 / 基准数据集 / 观点评论 |
| 代码与数据可用性 | 自动检测 + 链接验证 | "声明有"≠"实际可访问",分开标注 |
| 撤稿 / 勘误状态 | 撤稿数据库同步 | 卡片顶部红色标注 |

**B. 内容区(生成式摘要,但每条须锚定原文)**

- 研究背景 → 研究问题 → 核心方法 → 数据来源与实验设计 → 主要发现 → 创新点 → 局限性 → 与已有工作的区别 → 研究意义与应用价值。
- 生成规则:主要发现与局限性**必须**能定位到摘要或全文的具体段落;无法锚定的内容不得写入。

**C. 评估区(系统判断,显式标注为系统观点)**

- 六维评分(第 6 章)+ 一句话推荐理由 + "适合哪类读者"(按用户画像:本科生 / 硕博 / 资深研究者 / 产业研发)。
- 可信度判断:列出正负信号清单(样本量、基线、预注册、利益冲突声明、venue 预警状态等),不输出笼统的"可信 / 不可信"。

**D. 外部评价区(转述他人观点,须注明出处)**

- 引用语境(支持 / 反驳引用)、同行评论、媒体与政策关注、PubPeer 讨论(如有)。

### 8.2 三类陈述的强制区分(反幻觉核心规范)

所有生成文本必须归入且仅归入以下三类,并在渲染层用可视化标记区分:

| 类别 | 定义 | 标记 | 校验方式 |
|------|------|------|---------|
| **原文陈述** | 论文明确写出的结论 | 「论文称」 | 引用锚点(章节 / 句子级),抽检时可对照原文 |
| **系统推断** | 系统基于论文内容 + 外部数据得出的判断 | 「系统评估」 | 附特征与规则依据 |
| **第三方评价** | 其他论文、评论者、媒体的观点 | 「外部评价」 | 附出处链接 |

违规模式(禁止):把"作者未做但系统推测的适用场景"写进主要发现;把引用者的批评写成论文自己承认的局限。

### 8.3 分级摘要策略

按论文在报告中的地位分三档,控制成本与质量:

1. **重点推荐(每周 3–5 篇)**:全文(OA 可得时)解析 + 完整卡片 + 人工抽检队列。
2. **值得关注(每周 ≤15 篇,受 10.1 流量预算硬上限约束)**:摘要级卡片(B 区仅背景 / 问题 / 发现 / 局限四项)。
3. **收录可查(其余)**:仅书目区 + 一句话主题归类,不做生成式总结。

## 9. 研究空白与研究机会识别方法

### 9.1 研究空白的分类学(方法论地基)

学界对 research gap 有一条清晰的方法谱系,系统应直接采用而非自创分类:

- **Robinson, Saldanha & McKoy (2011, J Clin Epidemiol / AHRQ 报告)**:源于系统综述工程。两大部分:(a) 用 **PICOS 要素**(人群/干预/对照/结局/情境)刻画每个 gap;(b) **gap 成因四分类**——信息不足或不精确、信息有偏、结果不一致或一致性未知、信息不适切。成因分类与 GRADE 证据分级的域相互映射。来源:PMID 21937195;https://www.ncbi.nlm.nih.gov/books/NBK62478/
- **Müller-Bloch & Kranz (2015, ICIS)**:定性文献综述中严谨识别 gap 的六分类(矛盾证据 / 知识空白 / 行动—知识冲突 / 方法 / 评估空白 / 理论应用空白)。
- **Miles (2017)**:上述两者的七类合并版(证据 / 知识 / 实践—知识 / 方法 / 实证 / 理论 / 人群 gap),流传最广。**注意:这是博士生工作坊文稿而非同行评审论文**,引用时应说明。
- 2022–2026 年**没有**出现以"gap 识别方法"为主题的专门系统综述(经检索确认);最接近的是 LLM4SR(arXiv:2501.04306)与假设生成综述(arXiv:2504.05496)。

**系统采用方案**:以 Miles 七类为用户可见标签(易懂),内部映射回 Robinson 四类成因(可与证据强度挂钩)。

### 9.2 从论文文本自动抽取 gap 的技术现状

| 工作 | 任务 | 效果 | 对本系统的启示 |
|---|---|---|---|
| Lahav et al. 2022 (AAAI), arXiv:2108.13751 | 从全文抽取 challenge / direction 句 | PubMedBERT F1≈0.77–0.78;用户实验显著优于 PubMed 基线 | 句子级抽取是成熟可行的第一步 |
| 南理工团队 2023 (J. Informetrics), arXiv:2212.13860 | Future work 句识别 + 六类分型 | 识别 Macro-F1 90.7%;分类 SciBERT 加权 F1 72.6% | FWS 六分型可直接复用;代码数据公开 |
| LimGen 2024 (ECML-PKDD), arXiv:2403.15529 | LLM 生成论文 limitations | 4068 篇 ACL 论文数据集 | 金标准是作者自报 limitations,存在自报偏差 |
| LimitGen 2025, arXiv:2507.02694 | benchmark:LLM 能否发现论文关键局限 | **文献检索增强能显著提升**局限识别 | 系统的 gap 抽取必须带 RAG,裸 LLM 不够 |
| FutureGen 2025, arXiv:2503.16561 | RAG + LLM 反馈生成 future work | GPT-4o mini + 反馈最佳 | 反馈回路有效 |

**共性局限(设计约束)**:金标准依赖作者自报(自报偏差);评测多用 LLM-as-judge;领域集中在 NLP/生物医学;**裸 LLM 生成的 limitation 泛泛而谈,必须检索增强才能具体化**。另一个有意思的元发现:至今没有以方法学分类学(Miles 七类等)为标签体系的 gap 抽取语料——本系统积累的标注数据本身有学术价值。

### 9.3 系统综述工程中的证据缺口方法(跨学科可迁移)

- **Evidence Gap Map(3ie 方法论)**:构建**干预 × 结局矩阵**,把已有研究映射进单元格;区分**绝对缺口**(几乎无研究的单元格)与**综合缺口**(有原始研究但缺高质量综述)。这个矩阵思想可推广为任意领域的"方法 × 问题 / 数据 × 场景"缺口热图——是月报"研究空白"栏的理想可视化。来源:https://www.3ieimpact.org/evidence-hub/evidence-gap-maps
- **EPICOT 框架(Brown et al. 2006, BMJ)**:研究建议的结构化表述——Evidence 现状 / Population / Intervention / Comparison / Outcome / Time stamp,可选扩展疾病负担与研究设计。来源:PMC1602035
- **Cochrane Handbook 第 15.6.3 节**:禁止只说"需要更多研究",必须按 PICO 域逐结局说明所需研究的性质,并与 GRADE 证据确定性挂钩。

**迁移结论**:医学以外的学科同样适用其精神——系统输出的每个"研究空白"必须结构化(在什么对象上 / 用什么方法 / 缺什么证据 / 截至何时),拒绝"这个方向值得更多研究"式的空话。

### 9.4 假设生成的可靠性:实证基础与失败模式

**传统路线**:Literature-Based Discovery(Swanson ABC 模型,1986)——A 与 C 不共现但均与中介 B 共现 → 假设 A–C 关联;经典成功案例(鱼油–雷诺氏病、镁–偏头痛)后经临床证实。核心评估难题至今未解:无标准基准、"未被发现的知识"无法直接验证。

**LLM 生成研究 idea 的关键实证(必须写进系统设计)**:

- **Si, Yang & Hashimoto 2024(arXiv:2409.04109,79 位专家 298 份盲评)**:LLM idea 新颖性显著高于人类(5.64 vs 4.84),可行性略低;但 **diversity collapse** 严重——4000 个生成 idea 去重后仅约 200 个(5%);**LLM 自评估不可靠**(最佳 LLM 评审准确率 53.3%,低于人类间一致性 56.1%)。
- **Ideation-Execution Gap(Si et al. 2025, arXiv:2506.20803, ICLR 2026)**:43 位专家每人 100+ 小时真实执行这些 idea 后,LLM idea 全指标降幅显著大于人类 idea,**排名反转**——新颖性优势在执行后消失。
- **AI Scientist 独立评估(arXiv:2502.14297)**:引用中位数仅 5 条且陈旧、42% 实验因代码错误失败(12 个中 5 个)、**7 篇稿件中 4 篇(57%)含幻觉或错误数值**(已核实:全文原句 "Four of seven manuscripts (57%) contained incorrect or hallucinated numerical results")、把成熟技术误判为新颖。

**六大已证实失败模式**:幻觉引用、把已解决问题当 gap、可行性误判、idea 同质化、与已发表工作重复(知识泄漏)、LLM 自评估不可靠。

### 9.5 研究机会评估框架(交付核心)

设计原则:每个候选机会必须**可证伪、可溯源、可反驳**,把 9.4 的失败模式直接转成强制门禁。

**六维评估表(每个候选机会一份)**:

| 维度 | 证据要求(硬性) | 防护的失败模式 |
|---|---|---|
| ① 现有基础 | ≥3 条经 DOI/arXiv 解析验证的真实文献,至少 1 条为近 18 个月;标注检索日期与数据源 | 幻觉引用、文献感知过时 |
| ② 不足(gap) | 映射到 9.1 的 gap 分类 + PICOS/EPICOT 要素;**附"为何是真 gap"的反驳性检查**(记录"已检索未发现解法"的检索式与日期);"未搜到"≠gap | 把已解决问题当 gap |
| ③ 可行性 | 列资源清单(数据/算力/伦理审批)与获取路径;**禁止凭 ideation 评分断言可行** | ideation-execution gap |
| ④ 创新度 | 与最相似 N 篇**逐条差异比对**;标注新颖性来源(新问题/新方法/新数据/新组合) | 同质化、知识泄漏 |
| ⑤ 风险 | 撤稿检查、合规检查、抢发风险;标注置信度 | 数据污染 |
| ⑥ 适合人群 | 明确专长/团队规模/经费门槛,**并写明"不适合谁"**;是否可能成文、适合投稿的 venue 类型 | 过度承诺 |

**机会条目输出字段(与 10.4 模板对齐,v1.2 补全)**:研究问题 / 现有研究基础 / 当前不足(gap 类型标注)/ **为什么值得研究(为什么是现在)** / 可能的研究方法 / 所需数据 / 实施难度 / 创新程度 / 预期贡献 / 潜在风险 / 适合人群(含"不适合谁")/ 是否可能成文与适合投稿的 venue 类型。前六维评估表是这些字段的证据要求层。

**证据分级**:L1 已验证(DOI 可解析、元数据一致、非撤稿)/ L2 检索命中未逐条核实(标"待核")/ L3 模型推断——**禁止作为 gap 成立或可行性结论的唯一依据**。

**防止过度承诺的输出规范(强制)**:
1. 每个结论附置信度(高/中/低)+ 证据标签;无 L1 证据禁用"必然、突破性、空白"等绝对化措辞;
2. **引用零幻觉门禁**:所有引用输出前经存在性校验 + 撤稿校验,未通过者只进"待核清单";
3. gap 主动证伪:声明检索范围与"可能已被解决,未穷尽检索";
4. 新颖性评分不得转述为可行性或价值承诺;
5. 批量生成先去重并**报告去重率**;
6. 标明哪些环节 LLM 自动产出、哪些经人工核验;LLM 自评分不作最终排序唯一依据。

### 9.6 端到端空白识别流程(v1.2,补齐交付 6)

```
文本路线:limitations / future work / discussion / 综述与系统综述证据缺口 抽取(9.2 方法,带 RAG)
   ⊕
KG 路线:概念共现图 link prediction——高预测影响但从未共同研究的概念对(Impact4Cast,AUC>0.9,代码数据 MIT)
   ↓ 合并去重 → 候选 gap 池
   ↓ 检索验证:"真的无人做过吗?"(Owl 范式——用检索证伪,而非 LLM 断言;记录检索式与日期)
   ↓ LLM 扩写为机会条目(受 9.5 全部门禁约束)
   ↓ 排序校准:SciMuse 专家排序基准(100+ 位组长对 4,400+ 条 idea 的排序)离线校准排序器
   ↓ 输出:机会条目 + 穷尽度曲线(f = 1 − e^(−n/τ),Undermind 模型)量化"该方向已有多少前人工作"
   附加栏目:跨论文 claim 矛盾检测(ContraCrow 范式)供"争议"标签与月报共识/争议栏
```

作者维度扩展(二期):把作者节点加入概念超图(Sourati & Evans 2023,NHB)可区分"快被做掉的 gap"与"人类盲点 gap"。

## 10. 周报和月报设计

### 10.1 设计原则

1. **倒金字塔**:最重要的信息(本期必读 3 篇 + 一句话领域动态)放在最前,细节靠后;读者应能在 90 秒内读完摘要层。
2. **周报重"增量",月报重"结构"**:周报回答"这周发生了什么新的",月报回答"这个领域的格局发生了什么变化"。
3. **每条内容附证据与链接**,每个评分可展开看依据。
4. **显式的"本期覆盖声明"**:本期扫描了哪些源、多少篇、过滤掉多少、哪些源缺数——防止"没写=不存在"的误读;外部评价类信号(引用语境、社媒提及)计数为 0 时须附覆盖率提示("**缺失 ≠ 不存在**",scite 被第三方实测的教训,见 related_solutions 2.1)。
5. **流量预算(硬上限)**:每方向每周全报总量 ≤20 篇(必读 3–5 + 值得关注 ≤15),溢出论文顺延或归入"收录可查"——防警报疲劳的第一原则(Feedly 生产实践,见 related_solutions 2.2)。

### 10.2 周报模板

```markdown
# 【领域名】研究周报 · 第 N 期(YYYY-MM-DD ~ YYYY-MM-DD)

## 0. 本期速览(≤5 条,一句话一条)
## 1. 本期必读(3–5 篇,完整论文卡片)
## 2. 值得关注的新论文(≤15 篇,精简卡片,按子主题分组)
## 3. 快速升温的主题(本周提及量/预印本量突增的 1–3 个主题 + 证据)
## 4. 领域动态(会议截稿/录用结果、重要报告、撤稿与争议、数据集/工具发布)
## 5. 争议与讨论(有明显分歧引用或公开辩论的工作)
## 6. 短期值得跟进的问题(1–3 条,附"为什么是现在")
## 附录. 本期覆盖声明(扫描源、总量、过滤漏斗、已知缺口)
```

### 10.3 月报模板

```markdown
# 【领域名】研究月报 · YYYY 年 M 月

## 0. 本月综述(300 字以内的领域叙事)
## 1. 本月最重要成果 Top 5–10(完整卡片 + 入选理由)
## 2. 研究格局变化
   - 论文量与主题分布(对比上月/去年同期)
   - 研究热点排名(升/降/新进/退出)
   - 六类方向标签变动(成熟/快速增长/新兴/衰退/争议/疑似炒作)
## 3. 主要团队与机构动态(高产/新入场团队、代表作)
## 4. 方法与资源(本月重要新方法、数据集、基准、工具)
## 5. 共识与争议(正在形成的共识 / 悬而未决的争论,各附双方证据)
## 6. 研究空白与机会(2–5 条,按第 9 章机会框架输出,含风险标注)
## 7. 下月建议追踪(方向 + 触发事件,如"某会议将放榜")
## 附录. 数据与方法说明(覆盖声明 + 本月评分/聚类参数变动)
```

### 10.4 其余模板

- **论文卡片模板**:即 8.1 的四区结构。
- **趋势分析模板**:主题名 → 标签(六类之一)→ 关键指标(论文量曲线、引用加速度、顶会顶刊渗透率、团队多样性)→ 代表论文 3 篇 → 系统判读 → 不确定性说明。
- **研究机会模板**:见第 9 章机会评估框架(研究问题 / 现有基础 / 不足 / 为何值得 / 可能方法 / 所需数据 / 难度 / 创新度 / 预期贡献 / 风险 / 适合人群 / 成文可能与投稿去向)。
- **个性化推荐模板**:推荐论文 + 推荐理由(与用户兴趣模型的哪个成分匹配)+ 反馈按钮(有用 / 无关 / 已读过)。**负反馈采用归因式**:点"无关"须选原因(主题不符 / 不喜欢该来源 / 其他),选"其他"时**不更新模型**——只从有明确归因的信号学习(Feedly 模式);反馈回流兴趣模型(第 11、12 章)。

### 10.5 学科适配

- 医学 / 生物:周报增加"临床相关性"栏与证据等级(RCT / 队列 / 病例);争议栏关注安全性信号。
- CS:周报强化会议周期事件(截稿、放榜、oral/best paper);"代码可用"权重高。
- 数学 / 理论:降低周报频率的心理预期(可切换双周报),弱化"热度"类指标,强化"重要问题进展"叙事。
- 社科 / 政策:增加工作论文(SSRN/NBER 类)与政策引用栏。

## 11. 用户兴趣与学科分类体系

### 11.1 各方案对比

| 方案 | 优点 | 缺点 | 定位 |
|------|------|------|------|
| 固定学科分类 | 简单、冷启动快 | 粒度太粗,跨学科主题无处安放 | 仅作导航入口 |
| 树形学科分类(如 OpenAlex topics 层级) | 结构清晰、可继承参数 | 树是静态的,新兴方向滞后 | 作为**骨架** |
| 关键词订阅 | 精确、用户可控 | 召回脆弱(同义词/缩写)、噪声高 | 作为**精调手段** |
| 自然语言兴趣描述 | 门槛最低、表达力强 | 需要解析,存在理解偏差 | 作为**主入口** |
| 期刊/会议订阅 | 高置信、简单 | 只覆盖"已知的好来源" | 辅助信号 |
| 作者/机构订阅 | 追踪团队动向有效 | 覆盖面窄、有姓名消歧问题 | 辅助信号 |
| 论文种子扩展 | 冷启动质量高("给我 5 篇代表作") | 依赖引用图质量 | 强烈推荐的冷启动方式 |
| 语义向量兴趣模型 | 泛化好(持续学习由排序层的逻辑回归承担) | 不可解释、漂移风险 | 作为**召回底座** |
| 多标签分类 | 贴合真实兴趣(一人多题) | 管理复杂度 | 基本假设:兴趣=多个独立主题 |

### 11.2 推荐的混合方案:「主题档案(Topic Profile)」

用户的每个研究兴趣 = 一个独立的主题档案,档案内部是多层表示:

1. **锚定层**:自然语言兴趣描述(必填)+ 3–10 篇种子论文(强烈建议)→ 系统解析出学科挂载点(树形分类节点,可多个)。
2. **符号层**:自动生成 + 用户可编辑的检索式(关键词 / 同义词组 / MeSH 或领域受控词表)、订阅的 venue / 作者 / 机构清单。
3. **排序层(v1.2 裁决:采用 Scholar Inbox 配方,替代原"多质心向量层"设计)**:冻结的通用嵌入(GTE 类,见 7.3 选型)只做召回;排序交给**每个主题档案一个逻辑回归**——正样本 = 种子论文 + 收藏/点"有用",负样本 = 归因式"无关" + **随机采样数千篇未交互论文作 easy negatives**(防小样本边界外推失控);冷启动只需 5 正 + 3 负;每次反馈**秒级重训**。"一个兴趣的多个侧面"由线性模型对嵌入空间的判别自然覆盖,不再维护显式多质心。
4. **参数层**:从学科挂载点继承的默认参数(评分权重、venue 分级表、趋势阈值、报告栏目开关),用户可覆盖。

召回时并联进行:符号层检索式与订阅清单保**精确与定向**,排序层的冻结嵌入相似度保**泛化**;融合去重后由排序层逻辑回归定序(其输出即第 6 章相关性维,见 6.5 分工)。

### 11.3 兴趣模型的维护

- 每期报告的用户反馈(显式按钮 + 隐式点击)作为增量信号,触发排序层秒级重训;**归因门控**:仅从有明确归因的负反馈学习(见 10.4);符号层变更(加词 / 删词)必须经用户确认,保持可控感。
- 漂移保护:主动学习只请用户评"决策边界附近"的论文(信息增益最大);每月向用户展示"系统理解的你的兴趣"(关键词云 + 代表论文 + 画像周报),允许一键校正。

## 12. 自动化研究工作流(方法论层)

### 12.1 总流程

```
用户兴趣输入 → 主题档案构建(11 章)
   ↓
检索式生成 + 数据源路由(按学科选源,3 章)
   ↓
论文采集(增量,按日期游标) → 去重与版本合并(预印本↔正式版) → 元数据规范化(DOI/ROR/ORCID 对齐)
   ↓
venue 分级标注(4/5 章) → 相关性判断(粗排:向量+规则;精排:模型) → 质量/重要性评分(6 章)
   ↓
摘要/全文解析(仅入选论文,分级策略 8.3) → 主题聚类 → 趋势识别(7 章) → 论文总结(8 章) → 空白识别(9 章)
   ↓
报告草稿生成(10 章模板) → 质量门禁(自动校验:链接可达/引用锚定/撤稿检查) → 人工审核(抽检) → 发布推送
   ↓
用户反馈 → 兴趣模型更新(11.3) → 下一周期
```

### 12.2 各步骤的方法归属(规则 / 传统 ML / LLM / Embedding / 知识图谱 / 人工)

| 步骤 | 首选方法 | 理由与备注 |
|------|---------|-----------|
| 检索式生成 | LLM 生成 + 规则校验 + 用户确认 | LLM 擅长同义词扩展;检索式必须可见可编辑 |
| 数据源路由 | 规则 | 学科→源映射是稳定的配置 |
| 采集与增量 | 规则(游标 + 重试) | 确定性任务,不需要模型 |
| 去重/版本合并 | 规则(DOI/arXiv ID 精确匹配)+ 传统 ML(标题作者模糊匹配阈值) | 错误合并代价高,保守优先 |
| 元数据规范化 | 规则 + 权威 ID(DOI/ROR/ORCID) | 知识图谱的入口工序 |
| venue 分级 | 规则(查表:分级框架的映射表) | 表由第 4/5 章方法离线维护 |
| 相关性粗排 | Embedding 相似度 + 符号层命中 | 高吞吐、低成本;基于共享摄入湖本地匹配(2.3) |
| 相关性精排 / 质量评分 | **每档案逻辑回归(Scholar Inbox 配方,见 11.2)** + 传统 ML 特征模型 + LLM(少量文本维度) | 冷启动 5 正 3 负;可解释特征为主,LLM 输出仅作特征之一 |
| 主题聚类 | Embedding + 聚类算法 | 每期增量聚类 + 月度全量重聚 |
| 趋势识别 | 规则(指标阈值)+ 统计(突发检测) | 见第 7 章;LLM 仅用于给趋势命名与写叙事 |
| 论文总结 | LLM(带引用锚定约束)+ 规则校验 | 8.2 的三类陈述规范是硬约束 |
| 空白识别 | LLM 抽取(limitations / future work / **discussion** / 综述与系统综述证据缺口)⊕ **知识图谱 link prediction**(Impact4Cast 路线)+ 检索验证 | 见 9.6 端到端流程;单篇论文的说法不构成"空白" |
| 引用网络/合作网络 | 知识图谱 | 论文-作者-机构-venue-主题 五类节点 |
| 报告生成 | LLM(模板槽填充)+ 规则渲染 | 数字与链接由程序填,LLM 只写叙事段 |
| 质量门禁 | 规则(链接校验、撤稿比对、锚点存在性检查) | 全自动、一票否决 |
| 人工审核 | 人工(**共享池审核制**,v1.2) | 人审对象 = 学科**共享卡片池**(算一次审一次、全员复用,规模化不随用户数线性增长);重点推荐卡片 100% 过审;个性化层(排序/推荐理由)仅自动门禁 + 抽检;用户 flag 通道作补充审核信号(scite 双人复核模式) |

### 12.3 节奏设计

- **每日**:增量采集、去重入库、撤稿状态同步。
- **每周**:相关性排序 → 评分 → 周报流水线;周报生成后 24h 人审窗口。
- **每月**:主题指标重算与趋势标签更新(36 个月滑窗,见 7.7)、venue 分级表快照复查、月报流水线、兴趣模型体检报告。
- **每季/半年**:全量重聚类与新旧主题对齐(见 7.7);评分权重回测——**对上线前 1–3 年的历史论文回填"评分时点可得特征"做回测**(引用有 1–3 年滞后,当期论文无法即时回测;禁用评分时点之后的特征,防泄漏);分级框架更新(JCR/SCImago/CCF 目录更新;中科院分区已冻结,仅复查引用处标注)。

## 13. 技术架构建议

> v1.4 解冻:基于 2026-07-17 技术选型调研(版本/许可/价格均经一手来源核实,详见调研交付物),按 2.3 约束裁决。原则:**能不加组件就不加**——每个落选项都有明确理由。

### 13.1 两套推荐栈

**栈 A · 单用户零服务器版**($0/月固定成本;变动成本 = LLM 调用 + 季度嵌入 <$5):

```
GitHub Actions cron(错峰整点 + 游标自检 + keepalive commit)
 → Python 采集(增量游标;丢跑自动补齐)
 → SQLite(元数据 + FTS5 + sqlite-vec;单文件湖 = 可迁移资产)
 → DuckDB 1.4 LTS 季度 snapshot 批处理(OpenAlex 官方 Parquet 副本)
   + FAISS/rustworkx(聚类与图特征,回写主库)
 → GROBID crf + Docling(仅入选论文解析)
 → 裸 LLM SDK + 原生 structured output + pydantic(无编排框架)
 → Hugo → GitHub Pages(周报静态站)+ Resend 免费层(3000 封/月)邮件
```

**栈 B · 千用户共享湖版**(一台 16–64GB 单机起;新增固定成本 ≈ 服务器 + SES $2–5/月):

```
cron/systemd + PG 任务表(Procrastinate;DAG 复杂化后升 Prefect 3 / Dagster OSS 自托管)
 → 同一套采集代码 → PostgreSQL 18 + pgvector 0.8.5(共享湖 + 共享卡片池 + 行级档案隔离)
   + pg FTS/RRF 混合检索 → 批处理/解析/LLM 层与栈 A 完全相同
 → Hugo/Cloudflare Pages + Amazon SES($0.10/千封)
```

**迁移路径(只改部署拓扑不改 schema,满足 2.3 多租户约束)**:第 0 天起用 SQLite/PG 双方言兼容 schema;向量统一 float32(sqlite-vec blob ↔ pgvector 无损);检索层封装为薄模块(FTS5→tsvector 是唯一适配点);发送后端抽象 sender 接口(Resend→SES)。全程无推倒重来点。

### 13.2 关键裁决与理由(逐项)

| 选型项 | 裁决 | 落选者与理由 |
|---|---|---|
| 主数据库 | SQLite(A)/ **PostgreSQL 18 + pgvector 0.8.5**(B) | MySQL 系(生态弱);托管 PG 起步(违背零服务器) |
| 向量 | sqlite-vec → 升级位 LanceDB(A)/ pgvector(B) | **Qdrant/Milvus/Weaviate:1M 规模不值常驻服务**;Chroma(索引强制全驻内存,16GB≈360 万条封顶);Pinecone 免费层 2GB 装不下 1M×768d(需 ~3.6GB),付费 $20–50/月托底 |
| 搜索 | pg FTS/FTS5 + 向量 RRF | **ES/OpenSearch 不必要**:无 faceting/高 QPS/CJK 需求;排序不满意的增量补丁是 VectorChord-bm25/ParadeDB,仍不满意才考虑 OpenSearch |
| 图查询 | **不用图数据库**:主库边表 + 递归 CTE(2–3 跳,毫秒~百毫秒);全图算法季度批处理导出 rustworkx/igraph 回写 | Neo4j(JVM 常驻 + 数据双写,CE 无 RBAC 堵死多租户);**KùzuDB 已死(2025-10 归档,官网失效)**;Apache AGE(多跳性能 issue);duckpgq(研究性质,仅语法糖) |
| 调度 | GHA cron(A)/ cron + Procrastinate(B),复杂化后 Prefect 3(Apache-2.0)或 Dagster OSS | Airflow(运维件数最多);Temporal(持久执行引擎,错位);各家云托管(Prefect Cloud 免费层收紧、Dagster+ 无永久免费层) |
| 批处理 | **DuckDB 1.4 LTS + OpenAlex 官方 Parquet 副本**(s3 免费匿名下载,可按分区只取单学科) | Spark(单机 JVM 税);自建 JSONL→Parquet 管道(官方已提供 Parquet,此路线已被消灭) |
| PDF 解析 | **GROBID 0.9(Apache-2.0,CPU)引文/结构 + Docling(MIT)正文→Markdown**;GPU 预算时 MinerU 提质量 | marker(权重 OpenRAIL-M 有 $2M 营收门槛);pymupdf4llm(AGPL 灰区);Unstructured(重心转商业);**Science Parse/papermage 确认已死** |
| LLM 编排 | **裸官方 SDK + 原生 structured output(Pydantic schema)+ instructor/pydantic-ai** | LangGraph/Haystack(本项目无多步 agent 负载,为不存在的复杂度付抽象税);LlamaIndex(定位漂移向商业解析)。Pydantic schema 强制锚点/证据字段,与 8.2/15.3 反幻觉规范正向咬合 |
| 嵌入 | **sentence-transformers 5.6 + 本地模型**(默认 Qwen3-Embedding-0.6B(Apache-2.0),SPECTER2 作 A/B 对照);季度百万级全量租一张 4090 <$1 | 该规模成本不是决策变量(API 全家 <$50/百万篇),**决策变量是向量空间可复现性**(API 模型静默升级会破坏季度重算)与许可(jina-v3/v4、NV-Embed 非商用) |
| 渲染推送 | Hugo + GitHub Pages + Resend(A)/ + Amazon SES(B) | **SendGrid 免费层已取消**(仅 60 天试用);**MkDocs Material 已宣布 2026-11-05 EOL**,不可为新项目选型;自建 SMTP(送达率工程不值得) |

### 13.3 与 2.3 约束的对照验证

十条约束逐条核验全部 ✅(详见调研交付物 (c) 表)。两条新情报需记入:① **OpenAlex 每日变更文件已变付费项**——每日增量只能走 API 游标(与 2.3 既定路线一致,但断了"低价买变更文件"的后路);② 免费快照确认为季度更新且官方直接提供 **Parquet 副本**——季度重算的工程成本比预估更低。

## 14. 数据库与知识库设计

> v1.4 解冻。schema 以 SQLite/PG 双方言兼容 SQL 子集定义(SQLAlchemy/迁移工具生成两套 DDL)。

### 14.1 实体与关系(定稿)

**实体七类**:论文(paper)、作者(author)、机构(institution)、venue、主题(topic)、用户主题档案(profile)、报告(report)。

**关系**:引用(paper→paper)、版本(预印本↔正式版,保留历史)、隶属(author→institution)、发表于(paper→venue)、属于主题(paper→topic,带权重)、**报告—论文**(含卡片版本号)、**报告—更正事件**(支撑 15.1 撤稿追溯;报告发布后不可变,更正以事件追加)。

### 14.2 存储布局

- **主库**(SQLite 单文件 / PG):七类实体表 + 三张边表(citations、authorships、affiliations)+ 档案表(反馈明细独立表,行级归属用户,可整体导出——2.3 数据归属约束)+ 任务状态表(游标、预算计数)。
- **向量**:与主库同体(sqlite-vec / pgvector),float32,列:paper_id、model_version、vector——**model_version 必存**,季度重嵌时新旧空间不混用。
- **全文与解析产物**:OA PDF 与 TEI/Markdown 存对象存储(本地目录 / R2),库内只存路径与 hash;**重建摘要仅存内部处理区,不进任何对外表**(16.1 硬规则)。
- **批处理区**:DuckDB + Parquet(snapshot 分区、指标回算中间表),与主库物理分离;算好的百分位/趋势指标以"快照日期版本化"写回主库查表。
- **报告**:发布即写入不可变快照表(渲染后的 Markdown + 引用的卡片版本清单),更正通知走独立事件表。

### 14.3 索引要点

citations(src,dst) 双向索引(递归 CTE 的前提);papers 的 (pub_date, topic) 复合索引(增量扫描);FTS 索引(标题+自产摘要);HNSW 建索引放季度批处理窗口(pgvector 需 `maintenance_work_mem` 8GB 量级,错峰执行)。

## 15. 数据质量控制

### 15.1 撤稿与勘误同步(每日必做)

- **主源:Crossref/Retraction Watch 合并库**(2023-09 Crossref 收购后免费开放):GitLab CSV **每工作日更新**,20 字段含受控撤稿原因词表。**实测规模(2026-07-16)**:总计 71,128 条,其中撤稿 65,708、关注表达 3,581、更正 1,494、复职(Reinstatement)160,其余为其他更新类型。
- **不要只靠 Crossmark**:出版商自登记的 update 元数据会漏掉约 2/3 撤稿(收购时点 Crossref 自有 1.4 万 vs RW 4.3 万)。
- 交叉校验:PubMed PT="Retracted Publication" 双向链接;OpenAlex `is_retracted`。
- 落地:每日增量同步 → 命中论文在库内打红旗 → 已进过报告的论文触发**追溯更正通知**(参考 Zotero 2019 年起的内置做法)。
- **支撑机制(v1.2)**:维护"报告 ↔ 论文(含卡片版本)"反查索引;报告发布后为**不可变快照**,更正以附加通知形式送达(不改历史报告);对应第 14 章实体草案新增两条关系:报告—论文、报告—更正事件。

### 15.2 论文工厂与引用操纵防线

- **规模证据**:Wiley/Hindawi 事件累计撤稿超 11,300 篇、关闭 19 种期刊、年收入损失 $35–40M;出版商自估投稿中约 2% 疑似造假。
- **识别信号**(可编程接入):Problematic Paper Screener(9 类探测器:tortured phrases、SCIgen、Feet of Clay=引用撤稿文献等,每周扫 1.3 亿篇);PubPeer 命中;**偷塞引用**——实证发现某出版商 3 刊 DOI 元数据中 ≥9% 引用为正文不存在的注入引用(arXiv:2310.02192)→ 系统应抽样交叉核对正文引用与元数据引用;Clarivate 年度压制名单。
- 处理:可疑 venue / 爆发式引用降权(与第 5 章预警级联动),不做公开指控,只做内部降权与标注。

### 15.3 LLM 输出质量控制(有实证基线)

- **过度概括是最大风险**:10 个主流 LLM 在 26–73% 案例中把有限结论改写为普适断言,是人类的近 5 倍(OR=4.85);**"要求准确"的提示无效甚至适得其反**(Peters & Chin-Yee, RSOS 2025)→ 对策不是提示词,而是**结构化抽取 + 保留原文限定词(样本、条件、时态)+ 人工抽检**。
- **幻觉引用基线**:GPT-3.5 39.6% / GPT-4 28.6% / Bard 91.4%(Chelli et al., JMIR 2024);AI 搜索工具 >60% 回答错误(Tow Center 2025)→ 一切引用必须回链核验(与 8.2、9.5 的门禁一致)。
- 质量门禁清单(全自动、一票否决):引用存在性解析;撤稿比对;锚点存在性检查(生成句可定位到原文);链接可达性;数字一致性抽检(statcheck 类)。

### 15.4 去重与版本合并

- 主键:DOI + OpenAlex ID;精确匹配优先,标题+作者模糊匹配设保守阈值(错误合并代价高于漏合并)。
- **预印本↔正式版**:同一工作的两个版本必须合并为一条记录并保留版本历史(bioRxiv `pubs` 端点、Crossref 关系字段辅助);报告中引用正式版、标注预印本首发时间。

## 16. 版权、合规与伦理风险

### 16.1 摘要版权:核心结论

- 书目元数据(题名/作者/DOI)属事实,不受版权(美国 Feist 1991);**摘要是文字创作,通常受版权保护**(欧盟 Infopaq 案:11 词片段即可受保护)。
- **Crossref 官方立场**:元数据 CC0,但**摘要保留原发表许可**,再利用者须自行确认——不能因为"从 Crossref 拿到"就认为可再分发。
- **OpenAlex 的处理即行业范本**:不提供明文摘要,只给倒排索引(官方称因法律约束);注意倒排索引可无损重建,并非稳固豁免。
- **系统策略**:对外展示优先用①自产的生成式摘要(基于合规获取的内容)、②OA 许可明确允许的原文摘要、③Elsevier 类源遵守 ≤200 字符片段 + DOI 规则;订阅源摘要不公开二次分发(Scopus 政策明文禁止)。

### 16.2 TDM 法律例外速查

| 制度 | 范围 | 商业性 | opt-out | 合同可否排除 |
|---|---|---|---|---|
| 欧盟 DSM Art.3 | 科研/文化遗产机构 | 仅非营利科研 | 否 | **不可排除** |
| 欧盟 DSM Art.4 | 任何人、合法访问 | 可商业 | **权利人可机器可读保留**(Elsevier 已声明保留) | 可排除 |
| 英国 CDPA s.29A | 合法访问者 | 仅非商业研究 | 否 | 不可排除 |
| 美国 fair use | 个案四要素;索引/检索类使用有利判例(Google Books/HathiTrust);AI 训练判例分化中 | 视情形 | 无 | 独立于合同 |
| **中国** | **无 TDM 例外**;生成式 AI 办法要求训练数据"合法来源" | — | — | — |
| 日本 30-4 条 | 最宽松,含信息分析,可商业 | 可 | 无 | 学理有争议 |

**对本系统的含义**:①商业化部署不能依赖欧盟 Art.3 / 英国 s.29A(仅非商业);②须检测并尊重机器可读权利保留(Art.4(3),AI Act 第 53 条强化);③中国境内运营须格外保守——依托 API 授权与 CC0/OA 许可,而非法定例外;④只用合法获取语料(Bartz v. Anthropic:盗版库不属 fair use)。

### 16.4 用户上传论文的处理(v1.2 补,任务书 §11 要求)

- 用户上传的 PDF/全文**默认视为受版权保护**,仅用于该用户自己的解析、总结与档案匹配(私有处理);**不进入公共库、不进入共享卡片池、不用于任何模型训练**。
- 上传即视为用户声明其对该文献有合法访问权;系统不对外再分发原文,卡片对外只展示书目区 + 自产摘要。
- 解析产物(chunk、嵌入)与用户档案同生命周期:用户删除档案时一并删除;导出时随档案数据一起交付(与 2.3 用户数据归属约束一致)。
- 中国境内部署下(无 TDM 例外,16.2)对上传内容的处理尤须保守:仅限用户私有用途,不做任何跨用户复用。

- 推送精度:用户标记"有用"占比 ≥40%(对标 Scholar Inbox 类产品的可用线);

- 时效:论文公开到进入周报 ≤7 天。

## 20. 结论与推荐方案

**一句话方案**:以 OpenAlex⊕Crossref 为多学科主干、学科专用源(arXiv/PubMed 系)为增量入口,免费开放数据做底座;venue 分级用"五级规则引擎 + DORA 合规约束";论文评估用"六维可解释评分 + 可信度扣分制";趋势用"SPECTER2 聚类 + 七组指标 + 六类标签规则";空白识别用"gap 分类学 + 检索增强抽取 + 六维机会框架 + 零幻觉门禁";交付形式为带覆盖声明的周报/月报。

**五个最重要的调研结论**:
1. **数据层**:免费开放数据(OpenAlex/Crossref/PubMed/arXiv)足以支撑 MVP 及以后很长阶段;商业源只是增值层。但 OpenAlex 2026 信用额度制改变了"无限免费"的假设,采集预算需要精算。
2. **评价层**:2026 年是评价体系变局年(中科院分区停更、CCF 第七版、ICORE2026)——分级表必须做成带快照日期的可更新配置,且整个评价层受 DORA/Leiden/CoARA 合规约束。
3. **评估层**:LLM 在论文评估中"能用但必须戴镣铐"——过度概括率 26–73%、提示词无效、自评估准确率仅 53.3%,所以一切 LLM 输出都要过结构化门禁 + 引用锚定 + 人工抽检。
4. **机会层**:研究空白识别是差异化核心也是风险核心,ideation-execution gap 的实证(执行后排名反转)要求系统永远输出"有证据的候选 + 不确定性",绝不输出"确定可行的方向"。
5. **市场层**:15 款竞品无一同时具备"个性化推送 + 显式多维评分 + 自动周报月报 + 研究空白识别",定位空档真实存在;可信度/可复现性信号(撤稿库已免费、SciScore、statcheck)是最易建立的护城河。
