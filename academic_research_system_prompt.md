# 学术研究情报系统方法论调研提示词

你是一名负责学术情报、科研趋势分析和研究方法设计的高级研究助手。

我计划开发一个学术研究情报系统。该系统允许用户选择一个研究方向、学科领域或具体主题，然后持续追踪该方向的最新论文、重要研究成果、前沿趋势和潜在研究机会，并自动生成周报和月报。

请先不要直接开发系统。你的第一项任务是进行系统性的全网调研，并建立一份可用于后续产品设计和开发的研究方法论知识库。

请将最终成果整理为一个完整的 Markdown 文件：

`knowledgebase.md`

## 一、项目目标

该系统需要实现以下核心能力：

1. 用户可以选择学科、研究方向或具体研究主题。
2. 系统可以持续检索该方向最新发表或公开的论文。
3. 系统优先识别顶级期刊、顶级会议和高影响力研究机构的成果。
4. 系统能够判断论文的重要性、创新性、可信度和潜在影响。
5. 系统能够总结当前学界的主要研究方向、重要进展和研究争议。
6. 系统能够生成按周和按月更新的研究报告。
7. 系统能够从已有研究中识别尚未解决的问题、研究空白和潜在研究机会。
8. 系统能够向用户推荐值得深入阅读的论文，以及可以进一步开展研究的方向。
9. 系统应尽量适用于不同学科，而不是只适用于计算机科学。

请通过全网搜索、官方文档、学术数据库说明、大学图书馆资源、科研评价指南和已有学术情报产品，对以下问题进行研究。

- 更新速度

### 2. 顶级期刊和顶级会议识别方法

研究如何判断一个期刊、会议或论文是否具有较高学术影响力。

需要分析但不限于以下指标：

- Journal Impact Factor
- CiteScore
- SJR
- SNIP
- H index
- 期刊分区
- JCR 分区
- 中科院分区
- CORE Conference Ranking
- CCF 推荐会议和期刊
- Google Scholar Metrics
- 领域专家认可度
- 编辑委员会质量
- 录用率
- 引用情况
- 机构和作者影响力
- 论文获奖情况
- 社会和产业影响

请说明不同学科的评价方法为何不同。

例如：

- 计算机科学更重视会议。
- 医学、生物学和化学更重视期刊。
- 数学可能更重视长期影响而不是短期引用。
- 社会科学可能需要同时考虑期刊、工作论文和政策影响。

请建立一个跨学科的期刊和会议分级框架，例如：

- S 级：领域最具影响力的期刊或会议
- A 级：国际顶级或高影响力出版物
- B 级：领域内高质量出版物
- 普通级：经过同行评议但影响力有限
- 预警级：质量存疑、掠夺性或缺乏可靠同行评议

不要只依赖单一排名。

### 3. 论文重要性评估方法

建立一套论文评分框架，用于判断论文是否值得进入周报、月报或重点推荐列表。

建议考虑：

- 出版平台质量
- 作者及研究机构背景
- 研究问题的重要性
- 方法创新性
- 实验或证据质量
- 数据集规模和代表性
- 结果是否显著
- 是否有可靠基线比较
- 是否可复现
- 代码和数据是否公开
- 引用增长速度
- 社交和媒体关注度
- 产业应用潜力
- 临床或政策意义
- 与当前研究热点的关联程度
- 是否属于综述、指南、共识或基准论文
- 是否存在重大争议或方法缺陷

请设计一个可计算的多维评分模型。

至少包括：

- 质量评分
- 创新评分
- 影响力评分
- 可信度评分
- 相关性评分
- 时效性评分
- 综合推荐评分

同时说明不同学科应该如何调整权重。

### 4. 最新研究方向识别方法

研究如何从大量论文中识别当前学界正在形成的研究趋势。

请分析以下方法：

- 关键词频率变化
- 主题建模
- 语义向量聚类
- 文献共引分析
- 文献耦合分析
- 引用网络分析
- 作者合作网络分析
- 机构合作网络分析
- 关键词共现分析
- 突发词检测
- 引用增长率
- 论文数量增长率
- 预印本增长率
- 顶级期刊和会议的专题变化
- 研究基金项目变化
- 综述论文和观点论文分析
- 学术会议议题变化
- 产业研究机构发布内容
- 专利和论文联合分析

请设计一套趋势识别流程，区分：

- 成熟研究方向
- 快速增长方向
- 新兴方向
- 衰退方向
- 争议方向
- 可能被过度炒作的方向

### 5. 研究成果总结方法

研究如何自动生成高质量论文总结。

每篇重点论文的总结应至少包括：

- 论文标题
- 作者
- 作者机构
- 发表时间
- 期刊或会议
- DOI 或正式链接
- 论文类型
- 研究背景
- 研究问题
- 核心方法
- 数据来源
- 实验设计
- 主要发现
- 创新点
- 局限性
- 与已有工作的区别
- 研究意义
- 应用价值
- 可信度判断
- 是否有代码和数据
- 值得关注的原因
- 适合哪些用户阅读

需要明确区分：

- 论文原文明确陈述的结论
- 系统根据论文内容做出的推断
- 其他论文或评论者的评价

不得把推测写成论文原始结论。

### 6. 研究空白与未来研究方向识别

请重点研究如何从现有论文中推导出可供用户继续研究的问题。

分析以下信息来源：

- 论文 limitations 部分
- future work 部分
- discussion 部分
- 综述论文中的未解决问题
- 系统综述和 Meta analysis 的证据缺口
- 研究结果之间的冲突
- 数据集覆盖不足
- 样本规模不足
- 特定人群或地区研究不足
- 方法缺少外部验证
- 现有模型泛化能力不足
- 缺乏长期研究
- 缺少真实世界实验
- 缺少跨学科研究
- 缺少可复现代码和数据
- 理论和实践之间的差距
- 新技术尚未应用到该领域的机会

请设计一个研究机会评估框架。

每个潜在研究方向应包括：

- 研究问题
- 现有研究基础
- 当前不足
- 为什么值得研究
- 可能的研究方法
- 所需数据
- 实施难度
- 创新程度
- 预期贡献
- 潜在风险
- 适合本科生、硕士生、博士生还是专业团队
- 是否可能形成论文
- 适合投稿的期刊或会议类型

系统必须避免把缺乏证据的想法包装成确定可行的研究方向。

## 四、研究质量要求

1. 必须进行真实的互联网搜索，不能只依赖模型已有知识。
2. 优先使用官方资料、学术数据库官方文档、大学图书馆指南、出版社资料和同行评议论文。
3. 技术问题优先引用官方文档。
4. 学术评价问题应使用可靠的科研评价来源。
5. 所有可能随时间变化的信息必须核实当前状态。
6. 每个重要结论必须提供来源。
7. 不得虚构期刊、会议、论文、排名、API 或价格。
8. 对无法确认的信息必须明确标记。
9. 需要比较多个来源，避免只依赖单一评价体系。
10. 不要将影响因子等同于论文质量。
11. 必须考虑不同学科之间的差异。
12. 必须区分预印本、正式发表论文、综述、指南、共识和原始研究。
13. 对任何自动生成的研究机会，需要同时分析可行性和风险。
14. 最终文档应以方法论为主，而不是简单列出工具名单。

## 五、建议的工作方式

请按以下顺序开展工作：

1. 第一阶段：定义系统目标和边界
2. 第二阶段：调查论文数据源和学术评价体系
3. 第三阶段：建立论文质量与重要性评分框架
4. 第四阶段：设计趋势识别与研究机会发现方法
5. 第五阶段：设计周报和月报结构
6. 第六阶段：设计技术架构和自动化流程
7. 第七阶段：分析版权、数据质量和合规风险
8. 第八阶段：形成最小可行产品方案
9. 第九阶段：将全部结果整理为 `knowledgebase.md`
10. 第十阶段：检查所有引用、链接、结论和方法是否可靠
